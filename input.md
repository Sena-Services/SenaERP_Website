# Sena - AI Business Orchestrator

**Tagline:** An AI orchestrator that autonomously operates businesses on a custom ERP - designed and built on demand for small teams.

---

## 1. Big Bets / Assumptions

### State of the World
AI's effect on programming in mid-2025 will generalize across all desk work with high reliability by mid-2027. Even if scaling laws don't hold, companies will build more and more RL environments for specific use cases. We can have effects of AGI before AGI.

### AI Ecosystem
Agent Marketplace will emerge where we can create, train, publish and monetize specialized agents. Protocols/Infrastructure will develop for these agents to use tools and communicate with each other (MCP, A2A, etc).

### New ERPs
For the user, the AI becomes the only point of contact. Almost all information the user sees will be generated on demand by the AI. The ERP becomes the AI's operating layer, not an app for the user.

### Small Teams
Domain Specific Knowledge + AI Adoption = outsized leverage. Small nimble teams adopt AI faster. Barrier to entry for turning an idea into a business drops; rapid increase in count of solopreneurs/small teams. Cost spent on tokens will be the major indirect operation expense for these small teams.

**Thesis:** If above is true, small businesses will need a new kind of AI ERP that auto builds and runs itself end-to-end.

---

## 2. Existing Solutions

### Traditional ERPs + AI Integration
Three-tier Architecture (UI - Logic - Database) with AI workflows/agents added on top.

**Examples:** Odoo, Zoho, TallyPrime

### Composable Enterprise Platform
A new wave of Composable Enterprise Platforms which can create Interface, Database, Workflows and Agents via natural language + No-code tools.

**Examples:** Lindy.ai, Cofounder.co, Airtable, Zapier Platform, Relevance.ai, Retool, Softr, Tray.ai, Workato, Gumloop

## 3. Product Demo

### Builder

#### 1. Requirement Gathering
Understands user's business requirements via discovery/express stages.

#### 2. Building ERP
Trained to convert BRD into ERP Environment comprising of the following primitives:
- **UI:** Prebuilt Vue components + CDN artifacts
- **Data:** Frappe doctypes
- **Workflows:** Triggers and actions
- **Agents:** Agent development kit

#### 3. Registry Management
A configurable registry is maintained for ERP Environments and its primitives. The registry will have prebuilt templates that can be used as building blocks by the builder.

#### 4. On-Demand Generation
The builder is trained to generate on-demand environment primitives if requirements are not satisfiable using templates in registry.

### Orchestrator

#### 5. Live Mode
Once built, the application can be run much like existing ERPs.

## 4. Strategic Position

1. **Vision:** Sena's goal is to become a builder-orchestrator; abstracting the complex world of AI tools from the user.

2. **Focus:** Application layer experience with effective inference optimization rather than training AI to be good at any specific vertical/horizontal. The focus is on training the AI to build an environment for it to orchestrate a mesh of AI vendors optimally.

3. **Revenue Model:** Subscription + pass through cost with markup revenue model.

4. **Target Market:** Small teams; first-time ERP implementers; new AI-pilled startups.

## 5. Road Ahead

**Gradual Autonomy Path:** CEP (visible) → Hybrid (supervised) → BOP (autonomous)

### Phase 1: Composable Enterprise Platform (CEP)

Initially setup as an enterprise grade CEP using iPaaS for integration, pre-built UI templates, workflow registries and Frappe database structure. Build a registry and a community.

**Marketing Focus:**

**Feature-wise:**
- WhatsApp/Instagram/Email agents
- SoTA computer use
- Content creation (nano-banana/Sora2/Veo3, etc.)
- Talk To Your Tally (TTT)

**Industry-wise:**
- Travel agency
- DMCs
- Hotels
- Activity/Event Management
- Restaurants

### Phase 'n': Business Orchestration Platform (BOP)

Work in parallel on all challenges required to achieve the BOP vision:

#### 1. AI Capability Gaps
- Long-horizon planning (weeks/months) not production-ready
- Tool orchestration error rates too high for "invisible" automation

#### 2. Ecosystem Infrastructure Missing
- No agent marketplace with APIs to specialized vendors
- MCP/A2A protocols emerging but not mature
- Can't "plug into" specialized AI - each requires enterprise contracts

#### 3. Trust & Validation Required
- Users won't trust "invisible AI" without experiencing reliability first
- Need gradual autonomy: CEP (visible) → Hybrid (supervised) → BOP (autonomous)

#### 4. Operational Scale
- BOP support complexity requires 10+ person team (currently: 3)
- 24/7 debugging autonomous AI vs. CEP where users debug transparent workflows
- Gap: Grow team with CEP revenue before BOP support burden
